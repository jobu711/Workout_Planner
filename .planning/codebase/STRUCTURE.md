## Directory Purposes

**`.claude/agents/`:**
- Purpose: Store Claude AI agent role definitions for different task types
- Contains: Markdown files describing agent capabilities and preferred styles
- Key files:
  - `workout-tracker-frontend.md`: Primary agent for HTML/CSS/JavaScript modifications

**`.planning/codebase/`:**
- Purpose: GSD (getting stuff done) codebase analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md (as applicable)
- Generated by: `/gsd:map-codebase` orchestrator
- Consumed by: `/gsd:plan-phase` and `/gsd:execute-phase` for context

**`workouts/`:**
- Purpose: User-exported workout session files
- Contains: JSON files with workout data (date, routines, exercises, sets, weights, reps)
- Generated by: User clicking "Save Workout" button in Routines tab via `exportForIPhone()`
- Consumed by: User importing via file input in History tab, triggering `importWorkoutSession()`
- Format: Standard JSON structure with `appId: 'workout-planner-session'`, timestamp, routines array

## Naming Conventions

**Files:**
- Workout session exports: `[Routine_Name]_[Date].json` (e.g., `push_day_Jan_31.json`, `Shoulders_2026-02-02.json`)
- HTML file: singular lowercase with underscore (e.g., `workout_plan.html`)
- Directory: lowercase with underscore for multi-word (e.g., `workouts`, `.claude`)

**Directories:**
- Hidden directories: dot-prefix (`.claude`, `.planning`)
- User data: lowercase plural (e.g., `workouts/`)

**CSS Classes:**
- Kebab-case: `.exercise-card`, `.tab-nav`, `.routine-modal-overlay`, `.level-tag`
- BEM-inspired for nested components: `.exercise-details`, `.routine-modal-options`
- Color/level qualifiers: `.level-beginner`, `.level-intermediate`, `.level-advanced`, `.level-rehab`, `.level-warmup`
- Tab-specific: `.shoulder-card`, `.shoulder-tab`, `.shoulder-tab.active`
- Functional: `.active` (applied dynamically), `.show` (for notifications)

**JavaScript Variables:**
- Constants (immutable): `ROUTINES_STORAGE_KEY` (line 4859)
- Objects: `routines`, `currentExerciseData`, `workoutHistory`
- Functions: camelCase starting with verb (e.g., `showTab`, `loadRoutines`, `extractExerciseData`)
- IDs in loops: `routineId`, `exerciseId`, `timestamp`, `tabId`

**Data Attributes:**
- DOM elements: `data-tab="tab-[name]"` (e.g., `data-tab="tab-shoulder"`)
- Exercise data: Scraped from DOM via class selectors, no data attributes used for storage

**localStorage Keys:**
- `workout_routines`: Stringified JSON of `routines` object
- `workout_history`: Stringified JSON of `workoutHistory` array
- `plateau_dismissals`: Stringified JSON map of `{ exerciseName: dismissalTimestamp }`

## Where to Add New Code

**New Exercise (in existing tab):**
- Primary code: `workout_plan.html` within tab-content div (e.g., lines 1988-4770 for existing tabs)
- Structure: Create new `<div class="exercise-card">` with image, h3 title, description, details, muscles, form-tips
- Button injection: "Add to Routine" and "View History" buttons are auto-injected by `addRoutineButtons()` and `addHistoryButtons()` at initialization
- No additional JavaScript needed; buttons hook into existing modal system

**New Tab (exercise category):**
- HTML tab button: Add to `.tab-nav` (line 1971-1983) with unique class and data-tab attribute
- HTML content: Create new `<div id="tab-[name]" class="tab-content">` after `tab-history` (after line 4783)
- CSS styling:
  - Tab button active gradient: Add `.tab-[name].active { background: linear-gradient(...) }` (follows pattern at lines 1143-1157)
  - Card variant: If needed, add `.tab-[name]-card { ... }` similar to `.shoulder-card` (line 1218)
- JavaScript: No changes needed; `showTab()` works generically

**New Routine:**
- Data structure: Modify `routines` initialization (line 4865-4870) to include new routine ID
- HTML modal: Add new button in `.routine-modal-options` (lines 4790-4807) with onclick callback
- JavaScript: `renderAllRoutines()` (line 6193) will auto-render the new routine in the Routines tab
- localStorage: Automatically saved/loaded by existing save/load functions

**New Analytics Feature:**
- Calculation: Add new function similar to `getExerciseStats()` (lines 5572-5643) to aggregate data
- Rendering: Add new function to build HTML (similar to `renderPlateauAlerts()` lines 5829-5865)
- Integration: Call from `renderAnalytics()` (lines 5773-5790) to insert into analytics container

**Utilities and Helpers:**
- Shared utility functions: Add at top of script block, before `showTab()` (around line 4830)
- Example: `escapeHtml()` is at line 5202
- DOM helpers: Place near other DOM manipulation functions (e.g., near `renderRoutine()` at line 6149)
